# Governance

## Objetivo

Definir como se toman decisiones para mantener calidad, transparencia y continuidad del proyecto.

## Roles

- Maintainers: cuidan roadmap, revisiones y releases.
- Contributors: proponen y mejoran recursos via issues/PRs.
- Reviewers: validan calidad tecnica y claridad documental.

## Proceso de decision

- Cambios pequenos: decision por consenso en PR.
- Cambios estructurales: issue `proposal` + periodo de comentarios.
- En caso de desacuerdo: decide maintainer responsable explicando criterios.

## Principios de revision

- Priorizamos impacto real y mantenibilidad.
- Pedimos evidencia de uso o ejemplo reproducible.
- Preferimos recursos composables y no acoplados a una sola herramienta.

## Transparencia

- Decisiones relevantes se documentan en issues o PRs.
- Releases deben incluir changelog y notas de migracion si aplica.
