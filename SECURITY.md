# Security Policy

## Alcance

Este repositorio contiene recursos para agentes/prompting. Riesgos relevantes:

- Inclusiones accidentales de secretos.
- Patrones de uso inseguro o destructivo.
- Instrucciones ambiguas que puedan inducir acciones peligrosas.

## Reporte responsable

Si detectas un problema de seguridad:

1. No publiques detalles sensibles en issues publicos.
2. Abre un canal privado con maintainers (o issue marcado como confidencial si la plataforma lo permite).
3. Incluye pasos para reproducir, impacto y mitigacion propuesta.

## Buenas practicas minimas

- Nunca subir API keys, tokens o credenciales.
- Incluir advertencias para comandos destructivos.
- Evitar prompts que sugieran bypass de controles de seguridad.
- Documentar limites y supuestos de cada recurso.

## Divulgacion

Intentaremos confirmar recepcion en 72 horas y proponer plan de respuesta segun severidad.
