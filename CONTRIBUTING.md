# Contributing Guide

Gracias por contribuir a este proyecto.

## Antes de empezar

- Revisa issues abiertos para evitar duplicados.
- Si el cambio es grande, abre primero un issue tipo `proposal`.
- Mantiene PRs pequenos: idealmente una sola idea por PR.

## Flujo recomendado

1. Fork del repositorio.
2. Crea una rama descriptiva:
   - `feat/<short-name>`
   - `fix/<short-name>`
   - `docs/<short-name>`
3. Realiza cambios con foco y documentacion.
4. Abre PR usando el template.

## Convenciones

- Commits: Conventional Commits.
- Docs: Markdown claro, ejemplos concretos.
- Nombres de recursos:
  - `kebab-case` para carpetas y archivos.
  - Un recurso por archivo cuando sea posible.
- Compatibilidad: indicar a que herramientas aplica (`codex`, `claude-code`, `gemini-cli`, `opencode`, `generic`).

## Checklist de PR

- [ ] Describe problema y solucion.
- [ ] Incluye contexto de uso real.
- [ ] No contiene secretos ni datos sensibles.
- [ ] Documentacion actualizada.
- [ ] Riesgos/limitaciones explicitas (si aplica).

## Criterios de aceptacion

Un aporte suele aceptarse si:

- Resuelve un caso real y replicable.
- Tiene instrucciones de uso y limites claros.
- Mantiene consistencia con la estructura del repo.
- Mejora la calidad del ecosistema y no solo la cantidad.

## Codigo de conducta

Al participar aceptas `CODE_OF_CONDUCT.md`.
